## 4. 不符合点分析 ⚠️

### 4.1 ❌ 前端框架不一致（严重程度：⭐⭐⭐⭐⭐）

| 维度 | 原始需求 | 当前项目 | 影响 |
|------|---------|---------|------|
| **框架** | uni-app | React + Vite | 无法发布到小程序 |
| **多端支持** | 小程序 + H5 + APP | 仅 Web/H5 | 用户覆盖受限 |
| **路由系统** | uni-app 路由 | React Router | 需完全重写 |
| **组件库** | uni-ui | React 原生组件 | 需替换 |

**后果**：
- ❌ 无法发布到**微信小程序**（微信只认 uni-app/Taro）
- ❌ 无法发布到**抖音小程序**
- ⚠️ 打包成原生 APP 需要额外工具（Capacitor/Tauri）

---

### 4.2 ❌ 3D API 集成不匹配（严重程度：⭐⭐⭐⭐⭐）

| 维度 | 原始需求 | 当前项目 |
|------|---------|---------|
| **3D 生成 API** | 腾讯混元 3D / Replicate | **Gemini API**（Google 文本/图像 AI） |
| **认证方式** | TC3-HMAC-SHA256 / Token | Gemini API Key |
| **环境配置** | `HUNYUAN_SECRET_KEY` | `GEMINI_API_KEY` |

**问题代码**：
```typescript
// ❌ 当前项目（错误的 API）
// .env.example
GEMINI_API_KEY="MY_GEMINI_API_KEY"

// ✅ 项目需要的是
HUNYUAN_SECRET_ID="xxx"
HUNYUAN_SECRET_KEY="xxx"
REPLICATE_API_TOKEN="xxx"
```

**后果**：
- Gemini API **无法生成 3D 模型**（仅支持文本/图像理解）
- 需要完全重写 3D 生成逻辑
- 需要对接后端 API（当前项目无后端集成）

### 4.6 ❌ Three.js 在小程序端受限（严重程度：⭐⭐⭐）

| 平台 | Three.js 支持 | 3D 功能 | 视觉体验 |
|------|--------------|--------|---------|
| **React Web** | ✅ 完整支持 | 旋转/缩放/平移 | ⭐⭐⭐⭐⭐ |
| **uni-app H5** | ✅ 完整支持 | 旋转/缩放/平移 | ⭐⭐⭐⭐⭐ |
| **uni-app 小程序** | ❌ 不支持 | 需降级方案 | ⭐⭐ |

**小程序降级方案**：
- **方案 1**：`<model-viewer>` 组件（仅支持 glTF，基础旋转）
- **方案 2**：webview 嵌入 H5 页面（性能较差）
- **方案 3**：纯图片预览（无 3D 交互）

**后果**：
- 小程序用户体验**明显降级**
- 3D 预览核心功能在小程序端**无法完整实现**

### 6.2 核心难点

#### **难点 1：样式系统重写** ⭐⭐⭐⭐⭐

**改造工作量**：40-50 个 CSS 类需要重写

| 样式类型 | 数量 | 改造方式 | 工时 |

| CSS 变量 | 12 个 | 硬编码颜色值 | 2 小时 |

**改造示例**：
```css
/* ❌ React 版 */
.marq-content {
  display: inline-block;
  animation: marquee 15s linear infinite;
}
@keyframes marquee {
  0% { transform: translateX(0); }
  100% { transform: translateX(-50%); }
}

/* ✅ uni-app 版 */
/* 需要使用 uni-app 的动画 API 或 CSS（部分不支持） */
<view class="marq-content" :animation="animationData"></view>

<script>
// JS 驱动动画
const animation = uni.createAnimation({ duration: 15000 })
animation.translateX(-50).step()
this.animationData = animation.export()
</script>
```

### 7.2 具体视觉损失

#### **损失 1：CSS 变量主题系统失效** 🔴

```css

  --warm-sand: #F5F5F0;

.button {
  background: var(--warm-sand);
  color: var(--deep-charcoal);
  border-color: var(--slate-grey);

/* 一键切换深色模式 */
[data-theme="dark"] {
  --warm-sand: #1a1a1a;

/* ✅ uni-app 版（硬编码，难以维护） */

  background-color: #F5F5F0;
```

**影响**：
- 主题切换需要重写逻辑
- 颜色修改需要全局搜索替换
- 代码可维护性下降

### 9.2 关键决策问题
