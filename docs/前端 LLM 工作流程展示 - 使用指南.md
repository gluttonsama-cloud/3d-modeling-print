## 🎯 功能概述

前端 Agent 可视化页面现在支持**两种数据模式**：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **模拟数据** | 硬编码的演示动画 | 客户演示、离线展示 |
| **真实数据** | 后端 API 实时数据 | 生产环境、开发调试 |

---

## 🚀 快速开始

### Step 1: 启动后端服务

```bash
cd backend
npm run dev:mock
```

后端将在 `http://localhost:3001` 启动，并启用 Mock 数据库。

### Step 2: 启动前端服务

```bash
cd admin-web
npm run dev
```

前端将在 `http://localhost:5173` 启动。

### Step 3: 访问 Agent 可视化页面

打开浏览器访问：`http://localhost:5173/agents/visualization`

---

## 📊 功能说明

### 1. 数据源切换开关

**位置**: 页面右上角

- **模拟数据模式** 🟠
  - 显示"本地演示模式"标签
  - 展示"模拟场景运行"下拉菜单
  - 可播放 3 种预设场景动画

- **真实数据模式** 🟢
  - 显示"已连接后端 API"标签
  - 展示"触发真实决策"按钮
  - 实时接收 Socket.IO 事件推送

### 2. 模拟场景（模拟数据模式）

点击"模拟场景运行"，选择场景：

1. **正常订单流转** - 展示完整的订单处理流程
2. **库存不足触发采购** - 展示库存预警和采购流程
3. **设备故障自动重试** - 展示设备故障容错机制

### 3. 触发真实决策（真实数据模式）

点击"触发真实决策"按钮，将：

1. 调用后端 API 执行真实的 Agent 决策
2. 使用 LLM（DeepSeek）进行智能分析
3. 实时显示决策过程和结果

### 关键代码

#### 数据源切换

```typescript
const [useRealData, setUseRealData] = useState(false);

useEffect(() => {
  if (useRealData) {
    // 获取真实数据 + Socket.IO
    fetchRealDecisions();
    setupSocketIO();
  } else {
    // 清空数据，等待模拟
    setEvents([]);
  }
}, [useRealData]);
```

#### 实时事件推送

```typescript
socket.on('agent-event', (event: AgentEvent) => {
  setEvents((prev) => [event, ...prev]);
});

socket.on('agent-state-change', (state: any) => {
  updateAgentState(state.agentId, { 
    status: state.state === 'busy' ? 'processing' : 'idle'
  });
});
```

#### 触发真实决策

```typescript
await triggerAgentDecision('coordinator', 'evaluate_order', {
  orderId: `ORD-${Date.now()}`,
  customerName: '测试客户',
  material: '白色 PLA',
  volume: 50
});
```

## ⚠️ 注意事项

### 1. 后端依赖

**真实数据模式**需要后端运行：
- MongoDB/Mock DB
- Socket.IO 服务
- Agent 决策 API

**模拟数据模式**无需后端。

### 2. 环境变量

确保前端 `.env` 配置正确：

```env
VITE_API_BASE_URL=http://localhost:3001/api
VITE_SOCKET_SERVER=ws://localhost:3001
```

### 3. LLM 响应时间

真实决策会使用 LLM，响应时间：
- DeepSeek: 2-4 秒（推荐）
- GLM-5: 10-15 秒（可能超时）

## 🔧 故障排查

### 问题 1: 真实数据模式无数据显示

**检查**:
1. 后端是否启动（`http://localhost:3001/health`）
2. 浏览器 Console 是否有 API 错误
3. Socket.IO 是否连接成功

**解决**:
```bash
# 重启后端
cd backend
npm run dev:mock

# 查看后端日志
tail -f logs/app.log
```

### 问题 2: Socket.IO 连接失败

**检查**:
1. 后端 Socket.IO 配置
2. CORS 设置
3. 防火墙/代理

**解决**:
```javascript
// backend/src/app.js
const io = new Server(server, {
  cors: {
    origin: 'http://localhost:5173', // 前端地址
    methods: ['GET', 'POST']
  }
});
```

### 问题 3: 决策时间线不更新

**检查**:
1. Socket.IO 事件监听
2. `agent-event` 事件是否触发

**解决**:
```typescript
// 前端添加调试日志
socket.on('agent-event', (event) => {
  console.log('收到事件:', event);
  setEvents((prev) => [event, ...prev]);
});
```
