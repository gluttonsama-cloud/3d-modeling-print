#### 2.7 设备管理服务 - API 端点修复 (2026-03-09)
- **文件**: `admin-web/src/services/deviceService.ts`
- **修改原因**: 前后端 API 端点和 HTTP 方法不匹配，导致设备管理功能完全无法使用
- **修改内容**:

  2. `updateDevice`: HTTP 方法从 PUT 改为 PATCH（匹配后端定义）
  3. `updateDeviceStatus`: HTTP 方法从 PATCH 改为 PUT（匹配后端定义）
- **代码**:
```typescript
// 修复 1: 正确解析分页响应
export const getDevices = async (): Promise<Device[]> => {
  const response = await api.get('/devices');
  const data = response.data?.data || response.data || {};
  return data.items || data || [];  // 后端返回 data.items
};

// 修复 2: PUT → PATCH
export const updateDevice = async (id: string, data: Partial<Device>) => {
  const response = await api.patch(`/devices/${id}`, data);  // 原来是 api.put
  return response.data?.data || response.data;
};

// 修复 3: PATCH → PUT
export const updateDeviceStatus = async (id: string, status: Device['status']) => {
  const response = await api.put(`/devices/${id}/status`, { status });  // 原来是 api.patch
  return response;
};
```
- **状态**: ✅ **正式修复** - 前后端 API 对齐，应该保留
