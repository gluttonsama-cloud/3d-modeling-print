# 3D 打印多 Agent 管理系统 - 演示流程脚本

> 🎯 **演示时长**: 约 10 分钟
> 🌟 **核心亮点**: 多 Agent 协作、实时可视化、设备自动分配、库存智能管理

---

## 📋 演示准备清单

### 1. 启动服务（演示前 2 分钟）

```bash
# 终端 1：启动后端
cd backend
npm run dev

# 终端 2：启动前端
cd admin-web
npm run dev
```

**等待看到以下输出**:
```
[Agents] ✅ 所有 Agent 初始化完成
[Agents] 已注册: 协调 Agent, 调度 Agent, 库存 Agent
╔════════════════════════════════════════════════════════╗
║  Agents: ✅ 已初始化                          ║
╚════════════════════════════════════════════════════════╝
```

### 2. 打开浏览器

- 前端地址: **http://localhost:3000/agents**
- 后端健康检查: `http://localhost:3001/health`

## 🎬 演示流程

### 第一幕：系统概览（1 分钟）

**进入页面** → 默认显示 Agent 可视化页面

**讲解要点**:
> "这是一个 3D 打印农场智能管理系统，核心是**多 Agent 协作架构**。
> 
> 系统有 3 个智能 Agent：
> - **协调 Agent** - 决策中枢，负责订单处理和 Agent 协调
> - **调度 Agent** - 设备分配专家，智能调度打印机资源
> - **库存 Agent** - 材料管理，消耗预测和补货建议"

**页面亮点**:
- 左侧导航清晰展示所有功能模块
- 中央展示 Agent 协作图谱
- 三个 Agent 节点通过边连接，展示协作关系

---

### 第二幕：Agent 协作工作流（4 分钟）⭐ 核心亮点

**操作步骤**:

1. **切换到"真实数据"模式**（右上角开关）

2. **点击 "触发 Agent 协作" 按钮**

3. **观察 Agent 终端实时输出**:
   - 协调 Agent: 思考过程显示
   - 调度 Agent: 设备选择逻辑
   - 库存 Agent: 库存检查结果

4. **观察时间线**: 4 个事件按顺序出现

**讲解要点**:
> "现在我来展示系统的核心能力——**完整的 Agent 协作工作流**。
> 
> 点击按钮后，系统自动执行：
> 
> **步骤 1 - 协调 Agent 接收订单**
> - 分析订单信息
> - 确定材料需求
> - 路由到专业 Agent
> 
> **步骤 2 - 调度 Agent 分配设备**
> - 查询可用设备
> - 负载均衡评估
> - 自动分配最优设备
> - **实际更新设备状态**（可去设备管理页验证）
> 
> **步骤 3 - 库存 Agent 检查库存**
> - 计算材料需求量
> - 检查当前库存
> - 库存充足时自动扣减
> - 库存不足时发出警告
> 
> **步骤 4 - 协调 Agent 做出最终决策**
> - 综合评估结果
> - 生成决策建议
> - 返回置信度"

**技术亮点**:
> "每个 Agent 都有**独立的终端显示**，实时展示思考过程。
> 
> 边动画展示了 Agent 之间的**消息传递**。
> 
> 所有操作都是**真实的数据库操作**：
> - 设备状态自动更新为 'busy'
> - 库存数量自动扣减
> - 决策记录保存到日志"

---

### 第三幕：验证实际效果（2 分钟）

**操作步骤**:

1. **点击左侧 "设备管理" 菜单**
2. **验证**: 刚才分配的设备状态已变为 "打印中"

3. **点击左侧 "库存管理" 菜单**
4. **验证**: 对应材料库存已扣减

**讲解要点**:
> "Agent 的操作不是模拟，而是**真实的数据库操作**。
> 
> 你可以在设备管理页面看到：
> - 设备 PRINTER-002 状态变为 '打印中'
> - 当前任务显示订单号
> 
> 在库存管理页面可以看到：
> - 材料库存实时更新
> - 扣减记录可追溯"

## 🎯 演示总结

### 核心技术亮点

| 特性 | 说明 |
|------|------|
| 🤖 **多 Agent 协作** | CoordinatorAgent、SchedulerAgent、InventoryAgent 三 Agent 协同工作 |
| 📺 **实时终端展示** | 每个 Agent 的思考过程实时显示 |
| 🔗 **消息传递可视化** | 边动画展示 Agent 之间的通信 |
| 📊 **真实数据操作** | 设备分配、库存扣减都是真实数据库操作 |
| 🎬 **多场景演示** | 正常流转、库存不足、设备故障等场景 |

### 技术栈

- **前端**: React + Ant Design + ReactFlow + WebSocket
- **后端**: Node.js + Express
- **AI**: DeepSeek LLM (via 七牛云 API)
- **数据**: MongoDB + Redis (Mock 模式支持无依赖演示)

---

## 🚨 常见问题应对

### Q: Agent 初始化失败？
> A: 检查后端日志，确保看到 `[Agents] ✅ 所有 Agent 初始化完成`

### Q: 工作流执行报错？
> A: 确保后端服务在 3001 端口运行正常，检查 `/health` 端点

### Q: 设备/库存没有变化？
> A: Mock 模式下数据在内存中，重启后重置。刷新页面查看最新数据。
